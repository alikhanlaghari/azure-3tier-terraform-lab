# Phase 09 – Monitoring & Observability (Concept)

## Objective
Introduce monitoring and observability concepts for the deployed 3-tier architecture.

## Monitoring Strategy

### Infrastructure Monitoring
- Azure Monitor for resource health
- VM metrics (CPU, Memory, Disk)

### Logging
- Log Analytics workspace for centralized logging
- Collection of VM and network logs

### Network Monitoring
- NSG flow logs
- Load Balancer health probe monitoring

### Security Monitoring
- Azure Defender (concept)
- Bastion access tracking

## Future Improvements
- Enable Azure Monitor alerts
- Integrate Log Analytics workspace
- Create dashboards for system visibility

## Outcome
This phase outlines how the deployed infrastructure can be monitored and secured in a real production environment.
